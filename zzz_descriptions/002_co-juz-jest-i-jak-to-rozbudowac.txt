Co już masz i dlaczego to kluczowe
Higiena repo i gałęzi – czyste, zsynchronizowane branche, sensowne .gitignore.
→ stabilne środowisko do automatycznych agentów/CI.
Indeksowanie + metadane (indexer + blame enricher) – calls_functions, api_endpoints, ui_routes, wstępna złożoność, ostatni commit.
→ to jest surowy „wsad” do bogatszej IR i kontraktów.

→ wystarczy rozszerzyć payload i ewentualnie dołożyć kolekcje pomocnicze.
CLI/build scripts + testy – jednolity entrypoint, możliwość walidacji w CI.
→ baza pod „bramki jakości” (gates) i pełną automatyzację.
Jak to wpiąć w „docelowy” kierunek (bez wyrzucania)
Pomyśl o obecnych artefaktach jako warstwie surowych faktów, które wzbogacamy do kanonicznej IR:
Masz teraz	W IR (docelowo)	Co dodać/minimalna zmiana
filePath, module, kind, symbol, linie	Symbol, Route, Service, Template…	tylko mapowanie nazw + id stabilne
calls_functions	Call Graph (oddzielna kolekcja)	eksport krawędzi caller→callee
api_endpoints	API Contract (OpenAPI ref)	metoda/ścieżka/paramy; link do OpenAPI
ui_routes	UI Contract (route→component)	SSR/SSG decyzja, props/data contract
cyclomatic_complexity	Difficulty/criticality score	dorzuć churn, watchers, coupling
git_last_commit	History context	dodać churn (ile zmian/90 dni)
–	evidence/confidence	skąd dana cecha, z jaką pewnością
–	migration_target + invariants	wskazówki przepisu (component/hook/util)
Co ewentualnie poprawić (ale nie przepisywać od zera)
Evidence + confidence – do każdej cechy (endpoint/route/call) zapisz source i rule, plus confidence 0–1.
Wyodrębnij kolekcje: call_graph, api_calls, route_maps obok code_chunks.
API contract – z api_endpoints dorób {method, path, query/body keys} i generuj OpenAPI (min. 80%).
Criticality score – złącz cyclomatic + churn + „route exposure” (np. /checkout).
Walidator IR – JSON Schema + check w CI (spójność, brak NULLi, zakresy linii, unikalność id).
Minimalny plan integracji (działa na tym, co masz)

node scripts/js_html_indexer.mjs --root ./frontend-master --out runs/latest/chunks.mjs.jsonl

# 3) Konwersja → IR (nowy krok-adapter: doda evidence/confidence, migration_target=heurystycznie)

# 4) Walidacja IR (JSON Schema)
python3 scripts/validate_ir.py --schema schemas/ir.schema.json --in runs/latest/ir.jsonl
# 5) Ingest do Qdrant (code_chunks + call_graph/api_calls/route_maps)
python3 scripts/build_kb.py ingest --reset
