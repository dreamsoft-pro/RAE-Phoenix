Żeby dowieźć w pełni autonomiczną migrację AngularJS → Next.js, kluczowe jest przejście z „indeksu kodu” do modelu systemu z testami-oraklami, kontraktami i regułami przepisu. Poniżej masz listę fundamentalnych usprawnień, które realnie podniosą precyzję agentów (i od razu podpowiadają, co zbudować/zmierzyć). Wszystko jest pomyślane tak, by było deterministyczne, walidowane i łatwe do zautomatyzowania.
1) Kanoniczna IR (Intermediate Representation) aplikacji
Zamiast luźnych „chunków”, zdefiniuj spójny model domenowy AngularJS → Next/React:
Symbol table: Module, Controller/Component, Service/Factory, Directive, Filter, Route, Template, Asset.
Binding graph: skąd pochodzą dane w template (scope → prop), ile i jakich watcherów, liczba i złożoność wyrażeń ng-*, zagnieżdżenia ng-repeat.
Event graph: $scope.$on/$emit/$broadcast, $rootScope i ich przepływy.
Directive semantics: scope: {}, bindToController, transclude, require, compile/link → mapowanie na wzorce React (props/children/effect/ref/portal).
CSS & klasowe zależności: ng-class, ng-show/hide → reguły dla JSX/conditional rendering.
Ta IR jest wejściem do agentów (Planner/Refactor/UI/Test), a także do walidatorów (poniżej). Zadbaj o wersjonowanie schematu + walidator (JSON Schema) — każdy etap pipeline’u najpierw weryfikuje spójność IR.
2) „Kontrakty” zamiast heurystyk: API, I/O, UI
2.1 API Contract (automatyczny OpenAPI)
Z $http/$resource wyekstrahuj {method, url, pathParams, queryKeys, bodyKeys} + przykłady.
Zbuduj OpenAPI (min. 80% pokrycia) → generuj klienta TS (axios/fetch) + hooki React Query/SWR.
Agent Refaktor nie „zgaduje” – ma kontrakt do którego doprowadza kod.
2.2 I/O Contract funkcji/serwisów
Dla każdej funkcji: params, returns, throws, side-effects (I/O, global), purity flag.
Wygeneruj DTO TS (strict), które muszą zostać użyte w nowym kodzie. Agent Testów waliduje typowość w CI (tsc --noEmit).
2.3 UI Contract (Route/View)
route → component (stare) i app/(segment)/page.tsx (docelowe), lista props i danych wymaganych na stronie (SSR/SSG).
Reguły SEO: meta, canonical, i18n namespaces → testy orakle Lighthouse.
3) Deklaratywne „Recepty Migracyjne” (Migration Recipes)
Zamiast pisać ad-hoc transformacje, stwórz DSL na recepty (YAML/JSON):
pattern (Babel/ESTree, HTML AST, „template binding pattern”)
action (jscodeshift/transform: JS→TS, $http→fetch/axios, directive→<Component/>, filter→util)
postconditions (np. brak any, brak bezpośredniego DOM API, eslint --max-warnings=0, tsc --noEmit przechodzi)
preserve (zachowaj nazwy zmiennych/symboli)
rollback (jak cofnąć, jeśli testy padną)
Recepty są wersjonowane, idempotentne i możliwe do uruchomienia stukrotnie. Agent Refaktor wybiera recepty wg IR („directive z transclude=true” → „Portal/children” itd.).
4) Oracle testów — wielowarstwowe sprawdzanie równoważności
Autonomia bez orakli = loteria. Potrzebujesz kilku niezależnych źródeł prawdy:
Unit parity: testy funkcji/utili przed/po (Goldens).
DOM parity: snapshot porównujący drzewo DOM dla tych samych danych wejściowych (w AngularJS vs Next) – ignoruj atrybuty szumu; sprawdzaj strukturę i tekst.
Visual parity: Playwright screenshot diff z progiem (np. 0.1–0.3% różnicy); whitelist odchyleń.
Behavior parity: test e2e kluczowych scenariuszy (login, checkout) – eventy, nawigacja, requesty sieciowe (network assertions).
Perf/SEO: Lighthouse CI budżety (LCP/CLS/TTI), meta/opengraph, hreflang.
i18n parity: liczba kluczy per namespace, brak „gołych” stringów.
Agenci nie uznają zadania za zakończone dopóki wszystkie orakle nie są zielone.
5) Analiza „ryzyka migracji” i priorytetyzacja
Wprowadź Migration Difficulty Score per moduł/trasa:
complexity (cyclomatic + liczba binderów + poziomy zagnieżdżeń ng-repeat)
churn (częstotliwość zmian w Gicie)
eventfulness ($rootScope/events)
watchers (# i „ciężkość”)
coupling (stopień sprzężenia przez DI i DOM)
Wynik steruje kolejką zadań agenta (najpierw szybkie zwycięstwa, albo odwrotnie – P0 krytyczne ekrany).
6) Graf wywołań: statyczny + ślad runtime
Statyka jest świetna, ale AngularJS bywa dynamiczny. Dorzuć run-time trace:
Instrumentuj dev build (Chrome DevTools Protocol/puppeteer) → loguj XMLHttpRequest/fetch, route transitions, wywołania funkcji krytycznych (wrap z proxy) i zdarzenia.
Zlej to z grafem statycznym (IR). Każda relacja ma {source: 'static'|'runtime', confidence, evidence}.
Agent Planner ma wtedy „ślad użytkowy” (co faktycznie żyje), co usuwa masę szumu.
7) i18n — ekstrakcja i spójność
Parser stringów z template/JS (także {{ 'key' | translate }}) → centralny słownik (i18next format).
Mapowanie konkretnych miejsc użycia → namespace/klucz (dowód skąd).
Agent UI podczas przepisu nie wstawia literalnych stringów – tylko klucze.
8) Accessibility by-default
Linter a11y dla JSX, automatyczna kontrola aria-*, altów, kontrastu (axe-core).
Test orakl: brak krytycznych błędów axe w nowym ekranie.
9) Pipeline „hybrydy” i cut-over
Tryb mieszany (reverse proxy / subpaths) – część tras obsługują Next, reszta AngularJS; wspólna sesja, spójne cookie.
Każda migracja trasy to PR z: „patchem”, mapą routingu, wagą w kolejce, oraklami i rollbackiem.
10) Multi-agent z bramkami jakości (blackboard)
Zdefiniuj role i bramki (gates):
Planner → wybiera zestaw tras/komponentów na wejście (na bazie IR + MDS). Gate: kompletna IR + kontrakty.
Mapper → generuje recepty i plan transformacji. Gate: recepty kompilują się i przechodzą weryfikację postconditions.
Refactor → wykonuje codemods, JS→TS, hooki danych, JSX. Gate: eslint+tsc = 0 warnings/errors.
Test Synthesizer → generuje/paruje testy (unit/DOM/e2e). Gate: testy pokrywają min. zestaw zachowań.
Verifier → uruchamia orakle i budżety. Gate: wszystko zielone.
Integrator → przygotowuje PR, opis, artefakty, „how-to verify”.
Guardian → monitoruje regresje po merge (sentry/logi/web vitals). Gate: brak regresji.
Każda bramka zapina się na artefakty w repo (runs/<ts>/*.json, raporty, diffy) – to pozwala agentom wznawiać pracę deterministycznie.
11) Performance-first i memory discipline
Bounded concurrency wszędzie; stream I/O; zero masowych plików w RAM.
Batchowanie przy Qdrant; retry z jitter; timeouts.
Perf budżety w Next (raster domyślny, przeniesienie ciężaru do SSR/SSG gdzie sensowne).
Analityka Web Vitals + raporty w CI (przed/po migracji na trasach).
12) Dobre dane → lepsze decyzje agenta
Rozszerz metadata o:
evidence[] (reguła, plik, zakres linii)
confidence (0–1) na poziomie cechy (endpoint, call, route)
criticality_score i składniki
io_contract, api_contract_ref (link do wygenerowanego OpenAPI)
migration_target (React Component / Hook / Util / Layout / Route) + rationale
invariants[] (np. wymogi kolejności walidacji, niezmienniki biznesowe danej funkcji)
13) Warstwa dekompozycji UI (layout/design tokens)
Zbuduj taksonomię komponentów UI (atoms/molecules/organisms) i mapę reuse (ile miejsc używa dany fragment).
Automatyczny ekstraktor tokenów (kolory, typografia) → Tailwind config + shadcn.
Agent UI migruje nie tylko „jak jest”, ale normalizuje do design-systemu – drastycznie redukuje koszt utrzymania.
14) Safety & determinism
Reproducible builds (pinned versions, lockfiles).
Każdy krok: idempotentny i sygnowany run_id.
Twarde limity iteracji, cache wyników (anti-loop).
Sandbox (oddzielny workspace) dla transformacji; merge dopiero po zielonych bramkach.
15) Telemetria i feedback loop dla agentów
Zbieraj przyczyny porażek (lint/test/orakle) i utrzymuj bibliotekę „fix-up recipes”.
„Learning by doing”: gdy orakl X często pada – dodaj receptę, która temu zapobiega.
Ranking transformacji: który przepis daje najwyższy wskaźnik sukcesu dla danej klasy problemów (uczenie rankingowe offline).
Minimalny zestaw prac „na teraz” (żeby podbić skuteczność)
Ustal i wdroż IR + walidator (JSON Schema)
Dopisz evidence, confidence, io_contract, migration_target, criticality_score.
OpenAPI generator z $http → klient TS + hooki (React Query).
Recepty migracyjne (DSL) + silnik postconditions (eslint/tsc/test).
Orakle: DOM parity + Visual diff + Unit parity + Lighthouse.
Runtime tracing (puppeteer) → zlewka ze statyką w relacjach (confidence↑).
Queue & gates: w Jenkinsie/K8s – każdy agent pracuje na „zadaniu”, artefakty lądują w runs/<ts>/.
